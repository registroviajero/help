# Notification center

The **bell** in the admin panel is RegistroViajero's notification center. From here you see everything that has happened on your reservations, team, and accommodations without having to dig through them one by one.

## How to open it

Tap the **bell** icon in the top bar. The red number shows unread notifications.

## Structure

| Element | What it does |
|---|---|
| **List** | Latest notifications on top. Read ones appear greyed out. |
| **Type indicator** | Icon and colour by type (guest, Ministry, team). |
| **Summary** | Short text with the key info (which guest, which reservation). |
| **Time** | When it happened, in relative format (5 min ago, yesterday, etc.). |
| **Mark all as read** | Button at the top of the drawer. |

## Grouping

Notifications of the **same type** about the **same reservation** are grouped into a single entry. For example, if three guests on a reservation sign back-to-back:

- You see **one entry** that reads "3 guests have completed check-in".
- Tap the entry to see the breakdown of the three guests.
- Marking the entry as read marks all three at once.

If a more recent event comes in for the same reservation (for instance, a guest reopening their edits), the previous entries un-group so you can see the change.

## Tapping a notification

Tapping a notification takes you straight to the related resource:

- **Guest completed check-in** → guest's reservation.
- **Ministry confirmation / error** → reservation.
- **New team member** → team.
- **Reservation created** → reservation.

The notification is automatically marked as read.

## Notification center vs preferences

| Notification center | Preferences |
|---|---|
| Shows what happened | Decide which types you receive |
| Reading, grouping, marking | Toggle individual types on/off |
| Settings → Activity | Settings → Notifications |

If a type doesn't appear in the bell, first check in [Notifications](/en/guide/notifications) that it is enabled.

## Automatic cleanup

Notifications are automatically deleted after **30 days**, read or unread. That keeps the bell snappy without you having to empty it manually.

## Access per role

Each user has their own bell — notifications are **personal**. Teammates on the same agency see their own notifications, not yours. Preferences are also per user.
